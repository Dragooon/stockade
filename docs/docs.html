Complete configuration reference.

Agents

Each agent has a model, system prompt, tools, and optional permissions, container config, and sub-agents.

agents:
  main:
    model: claude-sonnet-4-6
    effort: high                  # low | medium | high | max
    system_mode: append           # append | replace
    system: |
      You are the orchestrator. Delegate to sub-agents.
    tools:
      - Bash
      - Read
      - Write
      - Edit
      - Glob
      - Grep
    subagents:
      - researcher
      - engineer
    sandboxed: true               # run in container
    container:
      memory: 2g
      cpus: 2.0
      # isolation: session        # per-scope containers (default: shared)
      # dockerfile: ./custom.Dockerfile
    memory:
      enabled: true
      autoDream: false
    permissions:
      - "deny:Write(/config/**)"
      - "allow:*"

Permissions

Per-agent, first-match-wins rules. Three actions: allow, deny, ask.

permissions:
  - "deny:Write(/config/**)"       # block config writes
  - "deny:Bash(rm *)"              # block rm commands
  - "allow:Bash(git *)"            # allow git
  - "allow:Read"                   # allow all reads
  - "ask:Write"                    # prompt user for other writes
  # no match → ask (HITL approval)

Selector formats

PatternMatches
*All tools, all invocations
ToolNameSpecific tool, all invocations
ToolName(glob)Tool with path or command glob

Path prefixes

PrefixResolves to
/Platform root (~/.stockade)
~/Home directory
./Agent working directory
//Absolute path

For Bash, the pattern is a glob matched against the command string. For file tools (Read, Write, Edit, Glob, Grep), it's a path glob.

Symlinks are resolved to canonical paths. .. traversal is normalized. Windows paths are case-insensitive.

Gatekeeper

Optional AI risk assessment for ask rules. Configured at the top level:

gatekeeper:
  enabled: true
  agent: reviewer              # any agent ID from your config
  auto_approve_risk: low       # auto-approve at or below this level

Risk levels: low, medium, high, critical. When a tool invocation matches an ask rule:

  1. Gatekeeper agent reviews the invocation
  2. Assigns risk level + summary + reasoning
  3. If risk ≤ threshold: auto-approve, notify channel
  4. If risk > threshold: present to user with review attached

RBAC

Role-based access control. Users are identified by platform identity (Discord ID, terminal username) and assigned roles.

rbac:
  roles:
    owner:
      permissions:
        - "agent:*"            # access all agents
        - "tool:*"             # use all tools
    user:
      permissions:
        - "agent:main"         # only talk to main agent
      deny:
        - "tool:Bash"          # can't use Bash
        - "tool:Write"         # can't use Write
        - "tool:Edit"          # can't use Edit

  users:
    alice:
      roles: [owner]
      identities:
        discord: "123456789012345678"
        terminal: "alice"
    bob:
      roles: [user]
      identities:
        discord: "987654321098765432"

Identity flows through the entire sub-agent chain. If Bob can't use Bash, that restriction applies even when the main agent delegates to the engineer agent on Bob's behalf.

Channels

Terminal

channels:
  terminal:
    enabled: true
    agent: main

Discord

channels:
  discord:
    enabled: true
    token: ${DISCORD_TOKEN}      # env var substitution
    bindings:
      - server: "SERVER_ID"
        agent: main
        channels: "*"            # all channels
      - server: "OTHER_SERVER"
        agent: researcher
        channels:                # specific channels only
          - "channel-id-1"
          - "channel-id-2"

Agents receive all messages in bound channels. The agent's system prompt determines when to respond vs stay silent (shared channel awareness).

Containers

Top-level container settings:

containers:
  network: stockade-net            # Docker network (created automatically)
  proxy_host: host.docker.internal # how containers reach the proxy
  port_range: [3001, 3099]         # port allocation range
  base_dockerfile: ./packages/worker/Dockerfile
  build_context: .
  health_check:
    interval_ms: 500
    timeout_ms: 30000
    retries: 3
  defaults:
    memory: 1g
    cpus: 1.0
  max_age_hours: 0                 # 0 = no auto-cleanup
  max_concurrent: 5
  proxy_ca_cert: ./data/proxy/ca.crt
  apw_path: ./packages/proxy/src/cli/apw

Per-agent container settings

agents:
  engineer:
    sandboxed: true
    container:
      memory: 4g                  # override default
      cpus: 4.0
      dockerfile: ./dockerfiles/coder.Dockerfile  # custom image
      isolation: session          # per-scope containers
      volumes:
        - /data/shared:/data:ro   # additional bind mounts

Image resolution

Three-level fallback: agent dockerfile → platform base_dockerfile → built-in worker Dockerfile. Images are auto-built on first use and rebuilt when the Dockerfile changes.

Credential Proxy

Configured in config/proxy.yaml:

Provider

CLI command templates for reading/writing secrets. {key} and {value} are substituted.

provider:
  read:  "cat config/secrets/{key}"
  write: "mkdir -p $(dirname config/secrets/{key}) && printf '%s' '{value}' > config/secrets/{key}"
  update: "printf '%s' '{value}' > config/secrets/{key}"
  cache_ttl: 60                   # seconds

Network Policy

First-match-wins. Controls what containers can reach.

policy:
  default: deny
  rules:
    - host: "api.anthropic.com"
      action: allow
    - host: "api.github.com"
      method: "GET"
      action: allow
    - host: "api.github.com"
      path: "/repos/*/pulls"
      method: "POST"
      action: allow
    - host: "api.github.com"
      action: deny
    - host: "*.googleapis.com"
      action: allow

Credential Routes

Map hosts to credentials. The proxy injects them into outbound requests.

routes:
  - host: "api.anthropic.com"
    credential: anthropic-api-key
    inject:
      header: x-api-key

  - host: "api.github.com"
    credential: github-token
    inject:
      header: authorization
      format: "Bearer {value}"

  - host: "api.tavily.com"
    credential: tavily-api-key
    inject:
      header: authorization
      format: "Bearer {value}"

TLS

The proxy auto-generates a CA cert on first run. Containers trust it via NODE_EXTRA_CA_CERTS.

tls:
  ca_cert: data/proxy/ca.crt     # auto-generated
  ca_key:  data/proxy/ca.key

Task Scheduler

Background task scheduling with cron, interval, or one-shot triggers:

scheduler:
  tasks:
    daily-report:
      agent: researcher
      prompt: "Generate a daily status report"
      schedule:
        type: cron
        expression: "0 9 * * *"    # 9am daily
    health-check:
      agent: main
      prompt: "Check system health"
      schedule:
        type: interval
        every_ms: 3600000          # hourly
