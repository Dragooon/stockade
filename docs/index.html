Stockade

Multi-agent orchestrator for Claude with layered security.

Alpha software. Architecture is stable and tested (749 tests), but APIs and config format may change.

Agents run in containers with no secrets, no direct internet, and per-tool permission rules. You poke precise holes when you need to.

Built on Claude Code

Stockade runs on the Anthropic Agent SDK — the same runtime that powers Claude Code. Each agent is a Claude Code session with access to built-in tools: Bash, Read, Write, Edit, Glob, Grep, WebSearch, WebFetch.

What Stockade adds:

  • Multi-agent orchestration — parent agents delegate via ask_agent MCP tool
  • Remote execution — Agent SDK runs inside containers, dispatched via HTTP
  • Permission layer — allow / deny / ask rules per agent, per tool, per path
  • Credential isolation — MITM proxy injects API keys at the network level
  • Session management — channel scopes mapped to SDK session IDs in SQLite

Security Layers

1. Container Isolation
Sandboxed agents run in Docker on an internal network. No direct internet.
2. Credential Proxy
MITM proxy strips auth headers and injects credentials per route. Agents never see API keys.
3. Tool Permissions (allow / deny / ask)
Per-agent rules with path globs: deny:Write(/config/**), allow:Bash(git *). No match = ask user.
4. Gatekeeper
AI-powered risk assessment for ask rules. Auto-approves low-risk, prompts user for higher.
5. RBAC
User roles control agent and tool access. Identity flows through sub-agent chains.
6. Network Policy
Deny-by-default allowlist per host, path, and HTTP method.
